# Privacy practices 回答シート

最終回答は実装・公開するプライバシーポリシーと完全に一致させます。GemMurmurはページ内容と閲覧中のURLを扱うため、**すべて端末内の処理でもデータ取扱いの開示が必要**です。

## 単一目的

閲覧中のWebページに、端末内で動くGemmaが生成する短いライブコメントを表示すること。

## データの取扱い

| Chrome Web Store上の区分 | 回答 | 実装上の根拠 | 外部送信 |
|---|---|---|---|
| Webサイトのコンテンツ | 扱う | タイトル、見出し、表示中・近傍のテキストをコメント生成に利用 | しない |
| 閲覧アクティビティ | 扱う | 現在のURL・ドメインを停止設定とプライベートページ判定に利用 | しない |
| ユーザー生成コンテンツ | 扱わない | コメントは拡張が生成し、ユーザー入力を収集しない | しない |
| 個人情報 | 扱わない | アカウント、氏名、メールアドレス等を収集しない | しない |
| 認証情報 | 扱わない | ログイン・パスワードページでは停止する | しない |
| 財務・健康・位置情報 | 扱わない | 収集・保存・送信しない | しない |

## 保存・共有・販売

- ページ本文、URL、操作履歴はGemMurmurのサーバーへ送信しない。
- ページ本文は永続保存しない。生成時の短命なメモリ内コンテキストとしてのみ扱う。
- 設定（ON/OFF、言語、停止対象、モデル状態）とモデルのキャッシュはブラウザ／端末内に保存する。
- 初回セットアップ時は、Hugging FaceからGemmaモデルファイルを取得する。この通信にページ内容や閲覧URLは含めない。
- データの販売、広告目的の利用、第三者への共有、ユーザープロファイリングを行わない。

## Dashboardでの認証文

> GemMurmurが扱うWebサイトコンテンツおよび閲覧アクティビティは、ユーザーに明示したライブコメント機能の提供だけに使用します。これらのデータを広告、販売、第三者提供、または人による閲覧に使用しません。ページデータは外部へ送信されません。

## 権限の説明

| 権限 / ホスト | ユーザー向け説明 |
|---|---|
| `storage` | コメント表示、言語、停止設定、モデル状態を端末内に保存します。 |
| `tabs` | 現在開いているページだけを特定し、ページ単位・サイト単位の停止操作に使います。 |
| `alarms` | 長時間利用時のコメント密度を管理します。 |
| `offscreen` | 表示を妨げずに端末内のGemmaモデルを読み込みます。 |
| `http://*/*`, `https://*/*` | 一般ページでコメントを表示し、ページ内容を端末内だけで処理します。ログイン・決済・メールなどでは自動停止します。 |
| Hugging Face関連ドメイン | 初回だけGemmaモデルを取得します。ページ内容は送信しません。 |
