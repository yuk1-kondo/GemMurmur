# GemMurmur 実機テストチェックリスト

バージョン: `package.json` の version とポップアップ右上の表示が一致していること。

読み込みフォルダ: `extension/dist`（`extension` ソース直下ではない）

---

## A. インストール・起動

- [ ] `npm run ci` が通る
- [ ] `chrome://extensions` で GemMurmur を読み込める（赤いエラーなし）
- [ ] ツールバーの GemMurmur アイコンからポップアップが開く
- [ ] ポップアップ右上に正しいバージョンが表示される
- [ ] モデル ready 後、Gemma コメントが流れる
- [ ] スクロール・停止・タブ復帰・マウス操作で操作反応コメントが出る
- [ ] 操作反応コメントは選択言語（ja/en/zh）に合わせて出る

## B. 設定 UI（最重要）

- [ ] コメントが流れている**最中**にポップアップを開ける
- [ ] ON/OFF トグルが反応する
- [ ] 一時停止 / 再開が効く
- [ ] 「このページでは停止」「このサイトでは停止」が効く
- [ ] 「再開」で元に戻る
- [ ] 言語切り替え（自動 / 日本語 / English / 中文）が反映される
- [ ] ページ上のクイック停止ボタン（一時停止 / 再開トグル）が効く

## C. モデル・WebGPU

- [ ] 「モデル読み込み」で進捗が表示される
- [ ] ダウンロード中も設定 UI が操作できる
- [ ] 完了後、状態が `ready` になる
- [ ] `ready` 後、ページ内容に沿ったコメントが増える（ルール雑談だけではない）
- [ ] WebGPU 非対応環境ではエラーメッセージが表示される
- [ ] 「削除」「再読み込み」「ダウンロード取消」が動く

## D. コメント体験

- [ ] コメントが右から左へ流れる
- [ ] スクロール・停止・タブ復帰などで操作反応コメントが出る
- [ ] 同じ文言がすぐ連発しない（重複抑制）
- [ ] 長時間滞在で密度が上がる（バズモード）
- [ ] Gemma 失敗時も、ページタイトル/見出しに触れたフォールバックが出る

## E. ページ種別（優先5種）

各ページで 2〜3 分滞在し、コメントがページ内容と無関係すぎないか確認。

| ページ種別 | 例 | OK |
|-----------|-----|-----|
| ニュース | NHK, Reuters など | [ ] |
| ブログ | note, Medium など | [ ] |
| 技術文書 | MDN, 公式ドキュメント | [ ] |
| Wikipedia | 任意の記事 | [ ] |
| GitHub | README があるリポジトリ | [ ] |

## F. 自動停止・プライバシー

- [ ] ログインページ（例: accounts.google.com）で自動停止または警告表示
- [ ] パスワード入力が主なページでコメントが流れない
- [ ] DevTools Network でページ本文が外部 API に送られていない（Hugging Face モデル DL のみ許容）

## G. パフォーマンス

- [ ] 通常の記事ページでスクロールがカクつかない
- [ ] 重い SPA（Twitter/X, Notion など）でもブラウザが固まらない
- [ ] タブを 10 個以上開いていても設定が押せる
- [ ] `chrome://extensions` → Service Worker → 検証 で赤エラーが大量に出ない

## H. 多言語

- [ ] 日本語ページ + 言語「自動」→ 日本語コメント
- [ ] 英語ページ + 言語「English」→ 英語コメント
- [ ] 中国語ページ + 簡体/繁体設定が反映される

## I. 既知の制限（失敗ではない）

- Google Docs / PDF / 動画プレイヤー内は未最適化
- モデル未読み込み時はルールベース中心
- モバイル Chrome は拡張機能を実行できないため対象外
- iOS SafariはSafari Web Extensionを含むiOSアプリとしてのパッケージ・署名・実機検証が別途必要

## J. モバイル移植前のUI確認

- [ ] 390px幅（iPhone標準相当）でポップアップに横スクロールがない
- [ ] 360px幅（Android標準相当）でポップアップに横スクロールがない
- [ ] タッチで切り替える操作をPointer Eventsで検知できる
- [ ] iOS Safari向けパッケージ後、実機でモデル取得・WebGPU・メモリ消費を確認する

---

## 問題が出たとき

1. `chrome://extensions` で GemMurmur を**更新**（または削除して `dist` を再読み込み）
2. Service Worker の「検証」でコンソールエラーを確認
3. ポップアップを右クリック → 検証 → Console を確認
4. バージョン・再現手順・URL をメモして報告

## メトリクス（任意）

DevTools Console で `murmurMetrics` または実装済みメトリクス名を確認:

- `rule_comment_latency_ms` — 初回コメントまで（目安 < 1s）
- `gemma_first_comment_latency_ms` — Gemma 初回（目安 < 15s）
- `buffer_underrun_rate` — コメント切れ（低いほど良い）
